# Harbor Release Template

## Задача на Максима / security review

**Проект:**  
`<название проекта>`

**Описание:**  
`<что делает сервис, какие типы импорта выполняет, где используется>`

**Репозиторий:**  
`<repo url>`

**Ветка / commit:**  
`<branch>` / `<commit sha>`

**Docker image:**  
`registry.i.bitrix24.ru/<project>/<service>:<tag>`

**Что изменилось:**  
- ускорение текущего импорта без смены бизнес-логики
- подготовка release/config артефактов
- локально проверено на тестовом портале

**Что проверить:**  
- корректность логики импорта
- безопасность обработки файлов и внешних вызовов
- отсутствие регрессий в queue/background сценариях
- готовность образа к Harbor publication

**Доступы:**  
- если нужен доступ к GitHub: `horokey`

**Release gate:**  
- Harbor scan без `Critical`
- Harbor scan без `High`

## Публикация в Harbor

1. Собрать образ.
2. Проставить новый tag.
3. Запушить образ в `registry.i.bitrix24.ru`.
4. Проверить vulnerability scan.
5. Этот же tag потом передать облачным администраторам.
