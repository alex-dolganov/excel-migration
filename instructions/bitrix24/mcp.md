# Инструкция для ИИ агентов: Bitrix24 MCP server

## Краткая сводка

**Bitrix24 MCP server** — сервер Model Context Protocol для работы с документацией Bitrix24 REST API. Он помогает быстро находить методы, параметры и примеры и использовать их в задачах интеграции.

**Основная ссылка:** https://apidocs.bitrix24.ru/sdk/mcp.html
**GitHub-документация:** https://github.com/bitrix-tools/b24-rest-docs/blob/main/sdk/mcp.md

---

## Когда использовать MCP server

- Поиск нужного метода REST API (название, область, назначение).
- Уточнение параметров и форматов полей.
- Получение примеров запроса/ответа.
- Быстрая проверка актуальности описания метода.

Если MCP server доступен в окружении, используйте его при любой работе с REST API.

---

## Как использовать в работе

1. Сформулируйте задачу (какой метод нужен и какие поля требуются).
2. Через MCP получите:
   - название метода,
   - перечень параметров,
   - пример запроса/ответа.
3. Сопоставьте результат с вашим кодом и SDK.
4. Дайте пользователю краткую инструкцию и ссылку на официальную страницу.

---

## Пример (мини‑диалог)

**Пользователь:** Какой метод нужен, чтобы получить список сделок с полями `ID` и `TITLE`?

**Ассистент:** Используй **Bitrix24 MCP server**, чтобы подтвердить параметры метода `crm.deal.list` и актуальный формат `select`. Официальная страница: https://apidocs.bitrix24.ru/sdk/mcp.html

---

## Важные правила

- Если MCP server недоступен, предложи пользователю подключить **Bitrix24 MCP server** и обязательно укажи ссылку https://apidocs.bitrix24.ru/sdk/mcp.html.
- Не дублируй большие фрагменты документации в ответе — достаточно ссылок и краткого резюме.
